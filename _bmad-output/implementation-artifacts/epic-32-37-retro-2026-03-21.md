## What Needs Improvement

### Parallel Batch Planning Too Aggressive
While parallelism works well, the batching was sometimes too aggressive — stories touching overlapping files or assuming schema changes from other in-flight stories. Need a pre-flight conflict check before launching parallel batches, and a setup step to ensure each story has what it needs.

### PR Verification Branch-Jumping Disrupts Design Sessions
PR verification switches branches in the main working tree, which disrupts Vincentlay's other Claude Code session used for design/ideation. Fixes pushed during verification can catch design work in progress. Need to isolate PR verification into its own worktree or clone.

### E2E Test Coverage Doesn't Grow
Existing E2E tests pass (PR verification ensures this), but they don't cover new pages, features, or user paths added in Epic 32/37. The PWYW flow, portrait states, shape library, homepage sections, and relationship CTA have no E2E coverage. Passing ≠ covering.

### Dev Agents Forgetting Migration Files
Dev agents would modify `schema.ts` without writing the corresponding migration SQL file. CLAUDE.md was recently updated to explicitly require migrations when schema changes. Compliance needs monitoring.

### Manual Testing as Primary Verification
Hands-on verification relied on manual spot-checking with the test user. Combined with E2E gaps, the safety net for new features is thin.

### Production CORS Issue
Production is deployed but has a CORS issue preventing active use. Not blocking development but must be fixed before real users.
