# Pain Point Categories

## Layout
Things in the wrong place, poor hierarchy, wasted space, unclear visual flow, inconsistent spacing, misaligned elements, poor responsive behavior.

**Questions to ask:**
- Is the visual hierarchy clear? Do your eyes know where to go first?
- Is space used efficiently? Any dead zones or cramped areas?
- Does the layout work at mobile width?
- Is above-the-fold content the most important content?

## Visual
Doesn't match the design identity, inconsistent styling, poor contrast, wrong typography scale, off-brand colors, missing dark mode support, inconsistent component usage.

**Questions to ask:**
- Does this feel like it belongs in this app?
- Are design tokens being used consistently?
- Does it work in both light and dark themes?
- Is the contrast sufficient for accessibility?

## Content
Wrong information hierarchy, missing or redundant content, unclear labels, poor copywriting, information overload, hidden important data.

**Questions to ask:**
- Is the most important information the most prominent?
- Is anything redundant that could be removed?
- Would a first-time user understand what they're looking at?
- If someone screenshots this, is it compelling and clear?

## Interaction
Confusing flows, missing feedback, accessibility gaps, unclear affordances, broken navigation patterns, missing loading/error states, poor animation/transition.

**Questions to ask:**
- Is it obvious what's clickable/tappable?
- Does the user get feedback for their actions?
- Can this be navigated with keyboard only?
- Are loading and error states handled?
